# source_data/ — 原始打印报表数据源

> 本目录归档 AutoPrint 自动打印任务的**原始导出报表（.xlsx）**，是线上仪表盘（`index.html` → Supabase）的上游数据来源。
> 仪表盘本身查询的是 Supabase，这些 xlsx 用于溯源、重新灌库、离线核对与审计。

---

## ⚠️ 数据关系与注意事项（重要）

这些文件**不是**干净的按月拆分，存在**重叠与增量补录**关系，请勿直接相加：

- **3 月数据**：`ReportAutoPrint_202603.xlsx`（主，20 行） + `ReportAutoPrint_202603_Updated.xlsx`（增量 3 行）+ `ReportAutoPrint_20260708_add.xlsx`（补录，含 3/27、3/30 及一条 **6/30** 数据）。
- **5 月数据**：`ReportAutoPrint_v202605.xlsx`（较完整，23 行）与 `ReportAutoPrint_v20260501.xlsx`（早期版本，21 行）内容高度重叠，**以 `v202605.xlsx` 为准**。
- **4 月数据**：`ReportAutoPrint_v202604.xlsx` 主覆盖 4 月，但含 1 条执行时间为 **2026-07-08** 的记录（疑为补录或录入异常），核对时需注意。
- **6 月数据**：仅 `ReportAutoPrint_20260708_add.xlsx` 中含一条 6/30 记录，无独立 6 月整月报表。

**建议**：需要全量统计时，以「主文件 + 增量/补录文件去重」的方式合并，或直接信任 Supabase 中的已灌库数据；本目录保留原始导出以便追溯。

## 📊 通用字段字典（Schema）

各文件列结构基本一致，仅表头语言不同（英文见 `(2)/(3)` 表，中文见 `(1)` 表）：

| 字段（英文） | 字段（中文） | 含义 |
|--------------|--------------|------|
| Title | 标题 | 任务标题。定时任务形如 `AutoPrint-2026-03-02T01:30:00Z`（含计划执行 UTC 时间）；手动任务为自定义名（如「每日报销单据打印」） |
| ID | — | 任务编号（部分导出含，部分不含） |
| 执行时间 | 执行时间 | 任务**实际执行时间**（本地时区，日期+时间） |
| 总数 | 总数 | 本次任务打印总份数 |
| 成功 | 成功 | 成功份数 |
| 跳过 | 跳过 | 跳过份数 |
| 失败 | 失败 | 失败份数 |
| 完成时间 | 完成时间 | 任务完成时间 |
| 附件Excel表格 | 附件Excel表格 | 是否生成附件 Excel（True/False） |
| 任务完成通知邮件 | 任务完成通知邮件 | 是否发送完成通知邮件（True/False） |
| Created | 创建时间 | 报表导出/记录创建时间 |
| 耗时（分钟） | 耗时（分钟） | 任务耗时（分钟） |
| 标签 | 标签 | 自定义标签（多为空） |

## 🔄 数据更新与重新灌库

1. 新增/补录报表：放入本目录，命名建议 `ReportAutoPrint_YYYYMM.xlsx`（整月）或 `_add.xlsx`（增量）。
2. 灌库至 Supabase：参考根目录 `SOP_supabase_data_upload.md`。
3. 灌库后线上仪表盘刷新即生效（Vercel 自动部署 + Supabase 实时查询）。
